Overview

The Challenge & Project Scope

Developed as part of the EECE.3170: Microprocessors I curriculum, this project focuses on low-level interaction with a physical microcontroller platform without an operating system layer[cite: 1]. The objective was to engineer an interactive, real-time two-player dice game requiring high timing accuracy, debounced switch signals, and custom multi-peripheral register configurations[cite: 1, 2, 23].

My Role

Developer and Designer – Responsible for writing the complete low-level driver initialization routines, structural interrupt vectors, pseudo-random generation logic via ADC sampling registers, hardware layouts, and signal processing loops[cite: 1, 2, 6].

Hardware Integration

Core Peripherals & Logic

The game system seamlessly orchestrates multiple distinct subsystems through direct memory-mapped register manipulation:

🎮

Dedicated Inputs

Players interact via dedicated tactile buttons (S1 and S2 mapped on Port B configuration registers)[cite: 2, 8].

⚡

Vector Interrupts

Port B utilizes Change Notification flags configured via hardware vector #9 to reliably trigger actions instantaneously[cite: 1, 6].

🔢

7-Segment Display

The resulting pseudorandom roll number is encoded dynamically and routed over Ports A and C outputs[cite: 3, 8].

🎲

ADC Randomization

An Analog-to-Digital Converter polls floating channel inputs to harvest noise, forming an unpredictable pseudo-random seed pool[cite: 1, 2, 14].

Implementation details

Low-Level Driver Configurations

The software design relies completely on direct hardware bit-masking and register manipulation rather than abstract APIs. Below is the primary initialization routine used to secure hardware pinouts and bind vector-mapped ISR labels globally[cite: 6, 8]:

📄 DiceGame.asm  ·  MIPS Initialization & Setup
## =========================================================================
## Interrupt Vector Configuration
## =========================================================================
    .globl  __vector_dispatch_9        ## Port B change notification = vector #9
    .section .vector_9, code, keep
    .align 2
    .set nomips16
    .ent __vector_dispatch_9
__vector_dispatch_9:
    j  isrvector9                      ## Jump to actual ISR code in text section
    nop
    .end __vector_dispatch_9

main:
    ## Configure I/O ports - LEDs and 7-Segment on Port A & C
    sw      zero, TRISA                ## TRISA = 0  --> all PORTA bits = output
    sw      zero, ANSELA               ## ANSELA = 0 --> all PORTA bits digital
    
    ## Configure Port B switches (pins 7 & 13) as digital inputs
    li      t0, _PORTB_RB7_MASK
    ori     t0, t0, _PORTB_RB13_MASK
    sw      t0, TRISB                  
    
    ## Configure RB2/AN4 for ADC generation seed
    li      t0, _PORTB_RB2_MASK
    sw      t0, TRISBSET               ## Ensure RB2/AN4 is set as input
    sw      t0, ANSELB                 ## ANSELB pin 2 = analog for ADC sampling

Deep-Dive Into the Subsystems

The program functions across two paradigms: an asymmetric background initialization stack, and an asynchronous, interrupt-driven runtime context[cite: 6, 21, 22].

  • Multi-Vectored Interrupt Architecture: To prevent polling constraints from reducing accuracy, the program implements the PIC32 multi-vectored interrupt model by modifying the INTCON, IPC2, and IEC0 registers[cite: 17, 18]. The __vector_dispatch_9 section explicitly maps Port B Change Notification directly to vector slot 9, letting hardware handle context routing seamlessly[cite: 6].
  • True Noise Sampling via ADC: To create an authentic and fair roll of a 6-sided die, register AD1CHS routes input channel 4 (AN4) into a 10-bit Analog-to-Digital Converter[cite: 12, 14]. By tracking an ungrounded, floating analog input pin, the conversion module harvests environmental thermodynamic voltage variants to seed the pseudo-random logic blocks dynamically[cite: 1, 14].
  • Hardware State Management: The state of the game is continuously tracked within a static register (s7), mapping states from active player turns to evaluating final metrics[cite: 15, 16]. This prevents button presses out of turn from modifying the display matrix.
📄 DiceGame.asm  ·  Interrupt Vector Handling & Debouncing
## =========================================================================
## Interrupt Service Routine (ISR) for Port B Input Triggers
## =========================================================================
    .global isrvector9
    .ent isrvector9
isrvector9:   
    lw      t8, CNFB                   ## Read the Change Notification Flag register for Port B
    
checkS1:
    and     t9, t8, s4
    beq     t9, zero, checkS2          ## If S1 flag bit is 0, skip checking S1
    nop
    
    sw      s4, CNFBCLR                ## Clear change notification flag for S1
    jal     delay                      ## Divert to Software Debounce Sequence
  • Asynchronous Edge Detection & Clears: When a user clicks a button, the change notification module sets a status bit inside the CNFB register[cite: 24, 25]. The ISR isolates this flag using a bitwise and instruction, clears the pending hardware request atomically through memory-mapped clear aliases (CNFBCLR), and jumps to state processing[cite: 25, 26].
  • Bare-Metal Software Debouncing: Mechanical tactile buttons exhibit minor physical contact ripples that cause a microchip to register multiple inputs from a single click. To address this, the code implements a discrete delay: subroutine block[cite: 23]. This runs a tight down-counting iterator loop through 25,000 cycles (0x61A8) to allow voltage states to stabilize before accepting subsequent inputs[cite: 23].

Outcome

Stable Hardware Execution

The MIPS assembly application successfully executes entirely within bare-metal memory structures[cite: 1, 6]. By implementing physical change notification stacks alongside a modular debounce delay loop, floating edge triggers were completely mitigated[cite: 1, 23]. The dice values shift uniformly, ensuring fair game evaluation and dedicated on-board LED notifications upon winning states[cite: 1, 4].
