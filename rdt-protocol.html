Overview

The Engineering Challenge

While standard internet protocols like TCP natively guarantee connection-oriented stability, configuring custom transport solutions over raw, connectionless pipelines presents critical synchronization challenges. This project details a custom **Reliable Data Transfer (RDT) Protocol Engine** written in Python that forces raw UDP socket flows to natively manage sequential stream tracking, acknowledgments, timeouts, and bit-error recovery loops.

System Capabilities

Designed and deployed a 5-staged architectural matrix tracking progressively complex transport states: from basic connection verification up to dynamic sliding window engines that safely mitigate simulated packet loss drops and payload byte inversions.

Bit Error Generation & Payload Verification

To ensure complete structural verification, the engine relies on logical bitwise XOR computations. This maps tracking values along an integrated header layout to verify accuracy upon arrival:

# Calculate parity validation frame through continuous XOR mapping
def calculate_checksum(sequenceNumber, data):
    checksum = sequenceNumber
    for byte in data:
        checksum ^= byte
    return checksum

# Inject random signal degradation by flipping individual bits within the payload
def flip_bit(data):
    bit_to_flip = random.randint(0, len(data) * 8 - 1)
    data_bytes = bytearray(data)
    byte_index = bit_to_flip // 8
    bit_index = bit_to_flip % 8
    data_bytes[byte_index] ^= (1 << bit_index)
    return bytes(data_bytes)

Deep Dive: Core Simulation Modules

  • Finite State Machine (FSM) Sockets: The engine encapsulates transmitter behaviors within explicit Finite State Machine blocks (senderFSM). It uses tracking variables to control structural base values, track upcoming sequence numbers, and monitor window limits continuously.
  • Timeout Retransmission Guardrails: Sockets are paired with non-blocking timing controls via sock.settimeout(). If a simulated packet loss drops a confirmation signal, the sender catches the exception, recovers gracefully, and immediately schedules a sequential resend of the unacknowledged window frame.
  • Robust Binary Packet Layout: Network payload components use a network-byte-order formatting mask ("!III" + str(dataSize) + "s"). This maps the Sequence Number, Window Size, and Checksum directly into a stable 12-byte binary header appended to each data chunk.
